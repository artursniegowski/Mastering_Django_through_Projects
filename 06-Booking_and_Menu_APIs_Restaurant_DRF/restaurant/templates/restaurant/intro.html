This is just the backend for the restaurant. This will cover two API endpoints. In order to use the API endpoints, users need to be authenticated and use their tokens!

API for browsing or adding menu items.
/api/menu/items
/api/menu/items/<int:pk>

API for making reservations or listing the bookings.
/api/bookings/tables
/api/bookings/tables/<int:pk>

Enpoints that are used for creating new users and obtaining the tokens.
/api/registration/
/api/registration/users/ -- for adding new users
/api/registration/token/login/ -- for obtaining the user token

Static content for the website.
/restaurant/

Examples:
http://127.0.0.1:8000/restaurant/
http://127.0.0.1:8000/api/registration/users/
http://127.0.0.1:8000/api/bookings/tables
http://127.0.0.1:8000/api/menu/items
